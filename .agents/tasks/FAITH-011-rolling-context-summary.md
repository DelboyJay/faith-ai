## Objective

Implement an adaptive summarisation system that monitors token usage within an agent's conversation history and triggers compaction when thresholds are exceeded. The `ContextSummariser` works alongside the `BaseAgent` from FAITH-010, automatically condensing older messages into a structured context summary persisted to disk. This prevents agents from exceeding model context windows during long-running tasks while preserving critical decisions and state.

```python
"""FAITH Rolling Context Summary & Compaction.

Monitors agent conversation token usage and triggers adaptive summarisation
when thresholds are exceeded. Summaries are persisted to context.md files
that survive container restarts.

FRS Reference: Section 3.5.2, 3.5.3
"""

from __future__ import annotations

import logging
from datetime import datetime, timezone
from pathlib import Path
from typing import Any, Optional

import yaml

from faith.utils.tokens import estimate_tokens

logger = logging.getLogger("faith.agent.summariser")

# Default model context windows (tokens)
MODEL_CONTEXT_WINDOWS: dict[str, int] = {
    "claude-sonnet-4-20250514": 200_000,
    "claude-haiku-235-20241022": 200_000,
    "claude-opus-4-20250514": 200_000,
    "gpt-4o": 128_000,
    "gpt-4o-mini": 128_000,
}

DEFAULT_CONTEXT_WINDOW = 200_000

# Defaults
DEFAULT_THRESHOLD_PCT = 50
DEFAULT_MAX_MESSAGES = 50
SAFETY_MARGIN_PCT = 10


class ContextSummariser:
    """Adaptive summarisation triggered by token count or message volume.

    Monitors an agent's conversation messages and triggers compaction when
    either the token count exceeds a percentage of the model's context
    window or the message count exceeds a configurable maximum. Summaries
    are generated by calling the agent's own LLM and persisted to a
    context.md file on disk.

    Attributes:
        agent_id: Unique identifier for the agent.
        model: The LLM model name the agent uses.
        threshold_pct: Percentage of context window that triggers summarisation.
        max_messages: Maximum message count before summarisation is forced.
        faith_dir: Root .faith directory path.
    """

    def __init__(
        self,
        agent_id: str,
        model: str,
        config: Optional[dict[str, Any]] = None,
        faith_dir: str = ".faith",
    ):
        self.agent_id = agent_id
        self.model = model
        self.faith_dir = Path(faith_dir)

        # Load per-agent config overrides
        agent_config = self._load_agent_config()
        user_config = config or {}

        # Priority: explicit config > agent config.yaml > defaults
        self.threshold_pct: int = (
            user_config.get("summary_threshold_pct")
            or agent_config.get("summary_threshold_pct")
            or DEFAULT_THRESHOLD_PCT
        )
        self.max_messages: int = (
            user_config.get("max_messages")
            or agent_config.get("max_messages")
            or DEFAULT_MAX_MESSAGES
        )

        logger.info(
            f"ContextSummariser initialised for agent '{agent_id}': "
            f"model={model}, threshold={self.threshold_pct}%, "
            f"max_messages={self.max_messages}"
        )

    # ──────────────────────────────────────────────────
    # Properties
    # ──────────────────────────────────────────────────

    @property
    def context_md_path(self) -> Path:
        """Path to the agent's context.md file.

        Returns:
            Path to `.faith/agents/{agent_id}/context.md`
        """
        return self.faith_dir / "agents" / self.agent_id / "context.md"

    # ──────────────────────────────────────────────────
    # Threshold checks
    # ──────────────────────────────────────────────────

    def should_summarise(self, messages: list[dict], model: Optional[str] = None) -> bool:
        """Check whether summarisation should be triggered.

        Summarisation triggers when EITHER condition is met:
        1. Token count exceeds (threshold_pct - safety_margin)% of the
           model's context window.
        2. Message count exceeds max_messages.

        The safety margin (10%) ensures summarisation kicks in before
        the hard limit is reached, giving room for the summarisation
        prompt itself.

        Args:
            messages: List of conversation message dicts.
            model: Optional model override (defaults to self.model).

        Returns:
            True if summarisation should be triggered.
        """
        if not messages:
            return False

        # Check message count threshold
        if len(messages) >= self.max_messages:
            logger.debug(
                f"Message count {len(messages)} >= max {self.max_messages}"
            )
            return True

        # Check token threshold with safety margin
        effective_model = model or self.model
        context_window = MODEL_CONTEXT_WINDOWS.get(
            effective_model, DEFAULT_CONTEXT_WINDOW
        )
        effective_threshold_pct = self.threshold_pct - SAFETY_MARGIN_PCT
        token_threshold = int(context_window * effective_threshold_pct / 100)

        total_tokens = self._count_message_tokens(messages)

        if total_tokens >= token_threshold:
            logger.debug(
                f"Token count {total_tokens} >= threshold {token_threshold} "
                f"({effective_threshold_pct}% of {context_window})"
            )
            return True

        return False

    # ──────────────────────────────────────────────────
    # Summarisation
    # ──────────────────────────────────────────────────

    async def summarise(
        self,
        messages: list[dict],
        existing_summary: str,
        llm_call: Any,
    ) -> str:
        """Generate a structured summary of messages using the agent's LLM.

        The summary captures key decisions, outstanding tasks, important
        facts, and current blockers. If an existing summary is provided,
        the LLM is instructed to merge new information into it.

        Args:
            messages: List of conversation message dicts to summarise.
            existing_summary: Previously generated summary (empty string if none).
            llm_call: Async callable that takes a prompt string and returns
                      the LLM's response string. This is the agent's own
                      LLM interface.

        Returns:
            The generated summary string in context.md format.
        """
        prompt = self._build_summary_prompt(messages, existing_summary)

        logger.info(
            f"Generating summary for {len(messages)} messages "
            f"(existing summary: {len(existing_summary)} chars)"
        )

        response = await llm_call(prompt)

        # Wrap in context.md format if the LLM didn't already
        if not response.strip().startswith("# Context Summary"):
            response = self._wrap_context_md(response)

        return response

    # ──────────────────────────────────────────────────
    # Context persistence
    # ──────────────────────────────────────────────────

    def load_context(self) -> str:
        """Load existing context.md content from disk.

        Returns:
            The file contents as a string, or empty string if the file
            does not exist.
        """
        path = self.context_md_path
        if path.exists():
            content = path.read_text(encoding="utf-8")
            logger.debug(f"Loaded context from {path} ({len(content)} chars)")
            return content
        return ""

    def save_context(self, summary: str) -> None:
        """Write the updated summary to context.md.

        Creates parent directories if they do not exist.

        Args:
            summary: The full context.md content to write.
        """
        path = self.context_md_path
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_text(summary, encoding="utf-8")
        logger.info(f"Saved context summary to {path} ({len(summary)} chars)")

    # ──────────────────────────────────────────────────
    # Compaction
    # ──────────────────────────────────────────────────

    async def compact_messages(
        self,
        messages: list[dict],
        llm_call: Any,
    ) -> tuple[list[dict], str]:
        """Compact a message list by summarising older messages.

        Splits messages into two halves. The older half is summarised
        (with disposable messages receiving minimal treatment). The newer
        half is retained verbatim. A system message containing the summary
        is prepended to the retained messages.

        Disposable messages (those with `disposable: true` in metadata)
        are replaced with a single-line summary rather than being fully
        summarised. This keeps tool output, status pings, and other
        ephemeral content from bloating the summary.

        Args:
            messages: Full list of conversation message dicts.
            llm_call: Async callable for LLM summarisation.

        Returns:
            Tuple of (retained_messages, new_summary) where
            retained_messages includes a system summary message at
            position 0.
        """
        if not messages:
            return messages, ""

        # Split: summarise the older half, keep the newer half
        split_point = len(messages) // 2
        to_summarise = messages[:split_point]
        to_retain = messages[split_point:]

        # Separate disposable and substantive messages
        substantive = []
        disposable_count = 0
        disposable_topics: list[str] = []

        for msg in to_summarise:
            metadata = msg.get("metadata", {})
            if metadata.get("disposable", False):
                disposable_count += 1
                # Extract a brief topic reference for the disposable message
                content = msg.get("content", "")
                topic = content[:80] + "..." if len(content) > 80 else content
                disposable_topics.append(topic)
            else:
                substantive.append(msg)

        # Load existing summary for context continuity
        existing_summary = self.load_context()

        # Generate summary from substantive messages
        if substantive:
            new_summary = await self.summarise(
                substantive, existing_summary, llm_call
            )
        else:
            new_summary = existing_summary

        # Append disposable summary line if any were compacted
        if disposable_count > 0:
            disposable_line = (
                f"\n\n### Compacted Disposable Messages\n"
                f"- {disposable_count} disposable message(s) removed during compaction."
            )
            new_summary = new_summary.rstrip() + disposable_line

        # Save to disk
        self.save_context(new_summary)

        # Build retained message list with summary preamble
        summary_message = {
            "role": "system",
            "content": (
                f"[Context Summary — compacted at "
                f"{datetime.now(timezone.utc).strftime('%Y-%m-%dT%H:%M:%SZ')}]\n\n"
                f"{new_summary}"
            ),
            "metadata": {"type": "context_summary", "disposable": False},
        }

        retained = [summary_message] + to_retain

        logger.info(
            f"Compacted {len(to_summarise)} messages into summary, "
            f"retained {len(to_retain)} messages "
            f"({disposable_count} disposable removed)"
        )

        return retained, new_summary

    # ──────────────────────────────────────────────────
    # Prompt construction
    # ──────────────────────────────────────────────────

    def _build_summary_prompt(
        self, messages: list[dict], existing_summary: str
    ) -> str:
        """Build the structured prompt sent to the LLM for summarisation.

        The prompt instructs the LLM to produce a summary in the context.md
        format defined in FRS 3.5.3, covering four categories: key decisions,
        outstanding tasks, important facts, and current blockers.

        Args:
            messages: Messages to summarise.
            existing_summary: Previously generated summary to merge with.

        Returns:
            The formatted prompt string.
        """
        # Format messages into a readable transcript
        transcript_lines = []
        for i, msg in enumerate(messages):
            role = msg.get("role", "unknown")
            content = msg.get("content", "")
            transcript_lines.append(f"[{i + 1}] {role}: {content}")

        transcript = "\n".join(transcript_lines)

        existing_section = ""
        if existing_summary.strip():
            existing_section = (
                "\n\n## Existing Summary (merge new information into this):\n"
                f"{existing_summary}"
            )

        prompt = f"""You are a context summariser for an AI agent. Your job is to produce a structured summary of the conversation so far.

## Instructions

Analyse the following conversation transcript and produce a summary in EXACTLY this format:

# Context Summary — Agent
## Last Updated: {{timestamp}}

### Key Decisions
- List each significant decision made during the conversation

### Outstanding Tasks
- List each task that is still pending or in progress

### Important Facts
- List facts, constraints, or context that would be important to remember

### Current Blockers
- List any blockers, issues, or problems that need resolution

## Rules
1. Be concise — each bullet point should be one line.
2. If merging with an existing summary, UPDATE existing items rather than duplicating them.
3. Remove items from Outstanding Tasks if they have been completed.
4. Preserve all Key Decisions — never drop a decision from the summary.
5. Output ONLY the formatted summary, no preamble or explanation.
{existing_section}

## Conversation Transcript:
{transcript}"""

        return prompt

    # ──────────────────────────────────────────────────
    # Internal helpers
    # ──────────────────────────────────────────────────

    def _count_message_tokens(self, messages: list[dict]) -> int:
        """Estimate total token count across all messages.

        Args:
            messages: List of message dicts with 'content' fields.

        Returns:
            Estimated total token count.
        """
        total = 0
        for msg in messages:
            content = msg.get("content", "")
            total += estimate_tokens(content, self.model)
        return total

    def _load_agent_config(self) -> dict[str, Any]:
        """Load per-agent config overrides from config.yaml.

        Reads `.faith/agents/{agent_id}/config.yaml` and returns the
        `context` section. Returns empty dict if file does not exist
        or has no context section.

        Expected format:
            context:
              summary_threshold_pct: 40
              max_messages: 30

        Returns:
            Dict of context configuration overrides.
        """
        config_path = self.faith_dir / "agents" / self.agent_id / "config.yaml"
        if not config_path.exists():
            return {}

        try:
            with open(config_path, "r", encoding="utf-8") as f:
                data = yaml.safe_load(f) or {}
            return data.get("context", {})
        except Exception as e:
            logger.warning(
                f"Failed to load agent config from {config_path}: {e}"
            )
            return {}

    def _wrap_context_md(self, content: str) -> str:
        """Wrap raw summary content in the context.md format.

        Args:
            content: Raw summary text from the LLM.

        Returns:
            Formatted context.md string.
        """
        timestamp = datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")
        return (
            f"# Context Summary — {self.agent_id}\n"
            f"## Last Updated: {timestamp}\n\n"
            f"{content}"
        )
```

### 2. Update `faith/agent/base.py` (FAITH-010)

Add summarisation hooks to the `BaseAgent` class. After each message is processed, check whether compaction is needed:

```python
# Add to BaseAgent.__init__():
from faith.agent.summariser import ContextSummariser

self.summariser = ContextSummariser(
    agent_id=self.agent_id,
    model=self.model,
    faith_dir=self.faith_dir,
)

# Add to BaseAgent — new method:
async def _check_and_summarise(self) -> None:
    """Check token usage and trigger summarisation if needed.

    Called after each message is processed. If summarisation is
    triggered, compacts messages in-place and publishes an event.
    """
    if not self.summariser.should_summarise(self.messages, self.model):
        return

    logger.info(f"Agent {self.agent_id}: summarisation triggered")

    retained, summary = await self.summariser.compact_messages(
        self.messages,
        llm_call=self._llm_call,
    )

    self.messages = retained

    # Publish event so PA knows summarisation occurred
    await self.event_publisher.publish(
        event_type="agent:context_summary_triggered",
        source=self.agent_id,
        channel=self.channel,
        data={
            "summary_length": len(summary),
            "retained_messages": len(retained),
        },
    )
```

## Integration Points

- **FAITH-010 (BaseAgent):** The `_check_and_summarise()` hook is called after each message is processed. The `BaseAgent` holds the `messages` list and `_llm_call` interface that the summariser depends on.
- **FAITH-008 (Event Publisher):** The `agent:context_summary_triggered` event is published via the agent's `EventPublisher` when compaction occurs. The PA can subscribe to this event to track which agents are hitting context limits.
- **`faith.utils.tokens.estimate_tokens()`:** Token counting utility from FAITH-010. The summariser imports this to estimate message token counts without calling the LLM API.
- **Per-agent config:** Reads `.faith/agents/{id}/config.yaml` (the same directory structure established in FAITH-001). Agents with long-running tasks can lower the threshold; agents handling short tasks can raise it.

---

## Acceptance Criteria

1. `ContextSummariser.should_summarise()` returns `True` when token count exceeds `(threshold_pct - 10%)` of the model's context window.
2. `ContextSummariser.should_summarise()` returns `True` when message count reaches `max_messages`, regardless of token count.
3. `compact_messages()` splits the message list, summarises the older half, and retains the newer half with a summary preamble.
4. Disposable messages (metadata `disposable: true`) receive a single-line summary note rather than full LLM summarisation.
5. `context.md` is written to `.faith/agents/{id}/context.md` in the format defined by FRS 3.5.3 (Key Decisions, Outstanding Tasks, Important Facts, Current Blockers).
6. Per-agent threshold overrides from `.faith/agents/{id}/config.yaml` are respected, with explicit constructor config taking highest priority.
7. `BaseAgent._check_and_summarise()` triggers compaction and publishes an `agent:context_summary_triggered` event when thresholds are exceeded.
8. All tests in `tests/test_context_summariser.py` pass (minimum 10 tests covering threshold logic, token triggering, message-count fallback, disposable handling, persistence, prompt generation, and format validation).

## Notes for Implementer

- `estimate_tokens()` is imported from `faith.utils.tokens` (defined in FAITH-010). If that module is not yet available, you can stub it as `len(text.split()) / 0.75` for development — the real implementation will be a thin wrapper around tiktoken or a model-specific tokeniser.
- The `llm_call` parameter to `summarise()` and `compact_messages()` is an async callable, not an LLM client object. This allows the BaseAgent to pass its own method (e.g. `self._llm_call`) which already has the correct model and credentials configured. The summariser never handles API keys or model selection directly.
- `context.md` is volatile state — it should be listed in `.gitignore`. It is regenerated from the conversation history whenever the agent restarts with messages, and its absence simply means no prior summary is available.
- The split point in `compact_messages()` is `len(messages) // 2`. This is a simple heuristic; future iterations may use a smarter boundary (e.g. split at the last user message before the midpoint) but the 50/50 split is sufficient for Phase 3.
- `_build_summary_prompt()` is a pure function (no I/O, no LLM calls). This makes it easy to test the prompt format independently of the summarisation pipeline.
- `MODEL_CONTEXT_WINDOWS` will need updating as new models are released. Consider moving this to a shared config in a later task. For now, the fallback `DEFAULT_CONTEXT_WINDOW` (200k) provides a safe default.
- The `_wrap_context_md()` method is a safety net for LLMs that don't follow the formatting instructions exactly. It only fires if the response doesn't start with `# Context Summary`.
